## Архітектура

Архітектура системи базується на таких принципах:

1. Модульність — ​​кожна функціональна область є окемим модулем
2. Розділення відповідальностей — чіткий розподіл обовʼязків
3. Dependency Injection
4. Масштабованість — систему можна розширювати, не впливаючи на існуючі функції
5. Інкапсуляція

Модульність надає наступні переваги:
1. Інкапсуляція — модуль є самодостатньою одиницею
2. Повторне використання — модулі можна легко використовувати повторно.
3. Тестованість — модулі легше тестувати
4. Командна робота — різні команди можуть працювати паралельно над різними модулями

Поділ на контролери та сервіси реалізує принцип Single Responsibility:
- Контролери - тонкий шар, відповідальний лише за взаємодію HTTP
- Сервіси містять всю бізнес-логіку та можуть використовуватися в різних контекстах

Dependency Injection надає наступні переваги:
- Слабка звʼязність
- Просте тестування
- Централізоване управління життєвим циклом об'єктів

## Структура проєкту

```bash
src/
├── modules/ # Основна бізнес-логіка
│ ├── users/ # Модуль користувачів
│ │ ├── users.controller.ts
│ │ ├── users.service.ts
│ │ └── users.module.ts
├── app.module.ts # Кореневий модуль
└── main.ts # Точка входу в додаток
```

```bash

npm run build

# Запуск тестів
npm run test

# E2E тестування

# Лінтинг коду
```
